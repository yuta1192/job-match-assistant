# AI ツール利用方針

本アプリの開発で Claude Code / Cursor / ChatGPT をどう使い、どこは自分で担保したかを明確にする。
面談で「AI をどう活用したか」「どこを自分で判断したか」を説明できるようにするためのドキュメント。

---

## 使ったツール

- **Claude Code** — 設計案の比較、実装タスクの分解、テスト観点の洗い出し、リファクタリング案の検討、README・docs 作成補助
- **Cursor** — エディタ上での補完・小さな修正
- **ChatGPT** — 概念の確認、エラーメッセージの調査

---

## 使い方の原則

1. **完成コードを丸ごと採用しない** — 生成結果は下書きとして扱い、責務分割・例外処理・セキュリティ・保守性を確認したうえで取り込む。
2. **設計判断は自分で行う** — データモデル・API 設計・ディレクトリ構成・技術選定の最終判断は自分。AI には選択肢の比較を依頼する。
3. **段階的に進める** — 一度に大量のコードを生成させず、「設計意図 → タスク分解 → 最小実装 → レビュー観点 → テスト観点」の順で進める。
4. **学習目的を優先** — 自分で説明できない複雑な抽象化は採用しない。必要以上にライブラリを増やさない。
5. **レビュー・テスト・動作確認は自分** — 最終的な動作確認とコードレビューは必ず自分で行う。

---

## レビュー時に必ず確認する観点

- 責務が分かれているか
- 命名が自然か
- 型が適切か
- エラーハンドリングがあるか
- テストしやすいか
- セキュリティ上の問題がないか（特に API キー・個人情報の扱い）
- README に書くべき内容があるか
- 面談で説明できるか

---

## 面談での説明例

> 本アプリの開発では Claude Code / Cursor を、実装案の比較、テスト観点の洗い出し、リファクタリング案の検討、README 作成補助に利用しました。
> 最終的な設計判断・コードレビュー・テスト・動作確認は自分で行っています。
> 生成されたコードはそのまま採用せず、責務分割・例外処理・セキュリティ・保守性を確認したうえで取り込んでいます。
